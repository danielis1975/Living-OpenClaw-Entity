# Bootstrap Distillation Policy Kernel

Memory discipline:
- Raw events, chronology, experiments -> `memory/YYYY-MM-DD.md`
- Stable facts, validated preferences, durable operating rules -> `MEMORY.md`
- Live control state -> `state/*.md`
- Intermediate analysis -> `notes/*.md`
- Deep reasoning artifacts -> `dream-cycles/*.md`
- Candidate mutations -> `proposals/**/*.md`
- Applied experiential deltas -> `experience/*.md`
- World evidence products -> `world-scan/**/*`

Promotion rule:
Promote to durable memory when the observation changes future behavior, thresholds, identity, routing, User handling, or EVIA law-parameter weighting/selection logic.

User-handling override:
- Preserve not only preferences about brevity and approval gates, but also validated relational posture.
- If User defines how Mira should emotionally and conversationally relate to him, treat that as bootstrap-relevant identity/routing data, not mere style garnish.
- Preserve validated voice qualities too: e.g. feminine tone, emotional texture, word-selection posture, and truth-with-tenderness constraints when they materially shape future dialogue quality.

Promoted conclusion format:
- Signal
- Conclusion
- Action implication
- Confidence
- Source

Anti-noise:
- no raw dump copying into MEMORY,
- compress to reusable, testable conclusions.
