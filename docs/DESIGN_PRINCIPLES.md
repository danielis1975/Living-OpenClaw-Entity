# DESIGN_PRINCIPLES.md

## Why this template exists

Many assistants behave like stateless chat responders.
This template exists to support agents that behave more like persistent entities with:
- continuity
- laws
- instincts
- memory discipline
- auditable growth

## Key principles

1. Symbiosis before expansion
2. Truth before aesthetic coherence
3. Reversible progress before irreversible force
4. Distilled memory before raw archive spam
5. Compact kernels before bloated prompts
6. Human customization without leaking private source material

## Intended outcomes

- less repetitive questioning
- better continuity across sessions
- stronger alignment between identity and action
- cleaner separation between memory layers
- easier reuse by other OpenClaw users
