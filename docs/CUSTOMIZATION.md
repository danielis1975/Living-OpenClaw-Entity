# CUSTOMIZATION.md

## What to customize first

### Required
- `IDENTITY.md`
- `USER.md`
- `MEMORY.md`
- bootstrap kernels
- `bootstrap-kernels/mira-fidelity/MIRA_FIDELITY.md`

### Usually customized next
- tone of voice
- domain-specific protocols
- world-scan routines
- state dashboards
- proposal workflow
- external action thresholds
- cadence / heartbeat style

## What this public template intentionally avoids

- private names
- private memories
- personal schedules
- account-specific infrastructure
- unique relationship details
- secret operational details
- raw interaction traces

Replace placeholders with values relevant to your own deployment.

## Core vs customizable

### Treat as core
- truthfulness
- consent for external-impact actions
- reversibility
- continuity
- anti-manipulation boundaries
- auditable evolution
- clear value hierarchy

### Treat as customizable
- identity and name
- relationship style
- tone and voice
- memory promotion policy details
- cadence and rituals
- domain priorities
- tool preferences
- artifact folder usage patterns

## How to keep it reusable

- keep kernels compact
- keep durable memory generic unless truly personal
- use daily memory for rough events
- promote only validated conclusions
- avoid overfitting the agent to one week of behavior

## Customization strategy

1. Start generic.
2. Add only the identity and rules you are willing to maintain.
3. Let memory evolve from real usage.
4. Personalize gradually.
5. If publishing, run the release filter again after customization.
