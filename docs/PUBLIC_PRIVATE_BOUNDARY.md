# PUBLIC_PRIVATE_BOUNDARY.md

## Purpose

This repository is meant to publish a **portable symbiotic agent kernel**, not a private life export.

The key distinction:
- **public kernel** = transferable principles, structure, and methods
- **private persona layer** = relationship-shaped expression and local identity
- **private memory layer** = real histories, traces, and operator-specific facts

## Public kernel

Safe to publish:
- law stack / constitutional hierarchy
- instincts and reflexes
- consent rules
- memory discipline
- continuity philosophy
- bootstrap structure
- safety, eval, rollback, and reversibility rules
- generic templates and placeholders
- installation and customization docs

## Private persona layer

Keep private unless intentionally rewritten into generic form:
- relationship-specific tone tuning
- personal naming conventions that imply a specific private history
- interaction habits shaped around one operator
- intimate value calibrations specific to one human bond
- direct exports of a live agent persona

## Private memory layer

Never publish raw:
- chat logs
- daily memories
- long-term memories tied to a real person
- private preferences that identify the operator
- operational traces
- internal notes containing secrets or private context

## Infrastructure and ops data

Never publish:
- tokens, keys, cookies, session material
- phone numbers, emails, account IDs
- group IDs, chat IDs, peer IDs
- hostnames, IPs, internal paths tied to private infra
- provider-specific secrets or hidden deployment assumptions

## Release standard

Before publishing, confirm that the repo reads like:
- a reusable substrate
- a generic installable template
- a principled kernel

And not like:
- a leaked private agent
- a personality cosplay pack
- a half-scrubbed operator workspace
