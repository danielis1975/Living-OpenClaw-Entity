# RELEASE_FILTER.md

Use this checklist before publishing or updating the public repo.

## Publish

### Core philosophy and kernel
- [ ] `AGENTS.md`
- [ ] `SOUL.md`
- [ ] generic `USER.md`
- [ ] generic `IDENTITY.md`
- [ ] generic `MEMORY.md`
- [ ] bootstrap kernels
- [ ] install / customization / memory docs
- [ ] public/private boundary docs
- [ ] safety / eval / rollback principles

### Generic structure
- [ ] `memory/README.md`
- [ ] empty placeholder folders for cognition artifacts
- [ ] comments or templates instead of personal content

## Rewrite before publish

- [ ] real names -> placeholders or generic examples
- [ ] relationship-specific language -> reusable wording
- [ ] operator-specific routines -> optional examples
- [ ] local paths -> neutral template paths
- [ ] references to specific projects -> generalized docs if needed
- [ ] strong claims about personhood -> honest framing about operating model

## Remove before publish

- [ ] secrets / tokens / auth material
- [ ] phone numbers / emails / account IDs
- [ ] chat IDs / group IDs / peer IDs
- [ ] hostnames / IPs / internal infra notes
- [ ] personal memory files
- [ ] exported conversation logs
- [ ] private notes tied to a real person
- [ ] accidental scratch files / malformed folders / old dumps

## Final review questions

- [ ] Does this look like a reusable kernel rather than somebody's leaked private agent?
- [ ] Can a stranger install it without inheriting private baggage?
- [ ] Are customization points obvious?
- [ ] Are safety boundaries and consent rules visible?
- [ ] Is the framing ambitious but still epistemically honest?
